Namenskonvention: Am Dateinamen hängende Größen wie …16q/24q/32q werden beim Vergleich abgeschnitten (Regex (\d{2,3})q$).

	werden intern angewandt (einmalig gecacht).

DPI-Faktor: Nur bei ##q-Funktionen: Optional dpiFaktor As Single = 1.
XML-Container: Kompakt und „neugierresistent“: GZip-komprimierte Payloads (Base64 in XML).
Warmup: Nicht nötig; du rufst einmal ImageContainer.LoadBasis(basis) beim Programmstart.
Stein-Verwaltung: Eigene XML, eigener Manager, laufzeit-reloadbar (nur nachladen & Cache leeren).

Wie das Hilfsprogramm die XMLs baut (Kurzleitfaden)

Basis-Pack (Integrierte):
Verzeichnisse unter …\Grafiken\Integrierte\<SatzX>\.

Jede Datei lesen:
Name in baseKey + optional sizeQ via StripSizeQ.
Wenn SVG: Kind="svg", DataGzBase64 = GZip(UTF8(svgText)).
Wenn PNG/BMP: Kind="png", DataGzBase64 = GZip(raw bytes).
SizeHint kannst du z.B. "16q" eintragen (rein informativ).
Ein Key pro Motiv (ohne ##q). Doppelte Keys → erlaubt (SVG+PNG), Auswahl erfolgt zur Laufzeit (SVG-Vorrang).

Stein-Pack (Mahjongsteine):
Root …\Grafiken\Mahjongsteine\<SteinSatz>\.
Darunter Ordner je SteinStatus (genauer Name = Enum-ToString).
In jedem Status-Ordner genau eine PNG pro SteinIndexEnum.
Prüfe Vollständigkeit/Überhang → Exception.
Prüfe Gleichheit aller Größen → RefWidth/RefHeight setzen, sonst Exception.
Schreibe SteinPack mit Entries = (Status, Index, GZip(Base64(PNG))).

Verwendung im Programm
' Beim Programmstart EINMAL:
ImageContainer.Instance.LoadBasis(AktGfxBasis.Satz1)

' Allgemeine Grafiken holen (z.B. Kompass):
Dim bmp1 As Bitmap = ImageContainer.Instance.GetBitmap("CompassN16q", 16, 16, dpiFaktor:=1.25F)

' Mahjong-Steine initialisieren:
SteinManager.Instance.LoadSteinSatz(SteinSatz.Klassisch)

' Steingröße:
Dim sz As Size = SteinManager.Instance.GetSteinSize()

' Konkreter Stein:
Dim s As Bitmap = SteinManager.Instance.GetStein(SteinStatus.ClickableOne, SteinIndexEnum.DracheR)

' Späterer Laufzeit-Wechsel des Satzes (nur Nachladen & Cache leeren):
SteinManager.Instance.LoadSteinSatz(SteinSatz.Modern)

Warum das zu deinen Punkten passt
Hierarchie SVG > Bitmap: umgesetzt; SVG wird mit Svg.NET gerendert.
##q-Suffix: beim Vergleich strikt entfernt, Größe separat verarbeitet; dpiFaktor nur dort wirksam.
Dark-Mode/Brighter: intern, zentral, ohne Zusatzparameter; Werte sind gepuffert und konstant.
Fallback-Größe: genau wie gefordert (100×100; 1:1-Logik).
„XML ist groß“: Nutzlast GZip-komprimiert und Base64-codiert → kompakt & robust.
Kein Warmup nötig: Nur LoadBasis beim Start, Stein-Sätze getrennt via SteinManager.
Steine getrennt: Eigene XML-Struktur, Reload zur Laufzeit möglich, Cache-clear eingebaut.
Ghost-/Reserve-Grafiken: werden bei fehlenden Einträgen gerendert, Größe = Referenzgröße.
